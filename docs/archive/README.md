# Docs Archive

这里放的是历史文档，不是当前 source of truth。

## 什么时候看归档

- 想追溯某个产品判断是怎么演进过来的
- 想回看某次设计或执行计划
- 想参考旧版本结构，但不确定它是否还适用

如果你是为了理解“现在仓库到底是什么”，先回到：

- `../README.md`
- `../product/canonical-product-blueprint-zh.md`
- `../deployment/vercel-demo.md`
- `../tech/system-architecture.md`

## 归档分类

### `product/`

旧产品稿、早期 PRD、阶段性 demo 说明、曾经有效但已被 blueprint 覆盖的说明文档。

### `tech/`

历史技术草案和中间层文档。它们有参考价值，但不一定还匹配当前依赖、代码结构或部署方式。

### `roadmap/`

旧范围文档和阶段性范围判断。

### `superpowers/`

按日期保存的历史设计稿和实现计划。保留决策痕迹，但不代表当前待办。

这些文档会保留当时的上下文和原始交叉引用。
也就是说，归档里的内部链接不一定全部跟随当前目录结构重写。
它们的价值主要是追溯历史，不是作为现行执行入口。
